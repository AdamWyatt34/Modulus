## Adding a New Source Generator

Copy this checklist and track progress:
- [ ] Step 1: Add `DiagnosticDescriptor` entries to `src/Modulus.Generators/DiagnosticDescriptors.cs` (use `MODGEN00X` IDs)
- [ ] Step 2: Create `src/Modulus.Generators/MyFeatureGenerator.cs` implementing `IIncrementalGenerator`
- [ ] Step 3: Write syntax predicate (no semantic model, no allocations)
- [ ] Step 4: Write semantic transform (returns equatable struct, not symbols)
- [ ] Step 5: Write `Execute` method using `StringBuilder` to emit source
- [ ] Step 6: Add tests in `tests/Modulus.Generators.Tests/MyFeatureGeneratorTests.cs`
- [ ] Step 7: Verify compilation succeeds with `dotnet build Modulus.slnx`

### Generator Skeleton

```csharp
[Generator]
public sealed class MyFeatureGenerator : IIncrementalGenerator
{
    public void Initialize(IncrementalGeneratorInitializationContext context)
    {
        var items = context.SyntaxProvider
            .CreateSyntaxProvider(
                predicate: static (node, _) => node is ClassDeclarationSyntax,
                transform: static (ctx, ct) => GetInfo(ctx, ct))
            .Where(static x => x is not null)
            .Collect();

        var combined = items.Combine(context.AnalyzerConfigOptionsProvider);
        context.RegisterSourceOutput(combined, static (spc, data) =>
            Execute(spc, data.Left, data.Right));
    }

    private static MyInfo? GetInfo(GeneratorSyntaxContext ctx, CancellationToken ct)
    {
        ct.ThrowIfCancellationRequested();
        if (ctx.SemanticModel.GetDeclaredSymbol(ctx.Node, ct) is not INamedTypeSymbol symbol)
            return null;
        // Validate, extract strings → return equatable struct
        return new MyInfo(symbol.ToDisplayString(), symbol.ContainingNamespace.ToDisplayString());
    }

    private static void Execute(
        SourceProductionContext spc,
        ImmutableArray<MyInfo> items,
        AnalyzerConfigOptionsProvider options)
    {
        if (items.IsEmpty) return;

        options.GlobalOptions.TryGetValue("build_property.RootNamespace", out var rootNs);
        rootNs ??= "GeneratedCode";

        var sb = new StringBuilder();
        sb.AppendLine("// <auto-generated/>");
        sb.AppendLine($"namespace {rootNs};");
        // ... emit code

        spc.AddSource("MyFeature.g.cs", sb.ToString());
    }
}
```

## Adding a New Analyzer + Code Fix

Copy this checklist:
- [ ] Step 1: Add `DiagnosticDescriptor` to `src/Modulus.Analyzers/DiagnosticDescriptors.cs` (use `MOD00X` IDs)
- [ ] Step 2: Create analyzer class with `[DiagnosticAnalyzer(LanguageNames.CSharp)]`
- [ ] Step 3: Override `SupportedDiagnostics` to return your descriptor
- [ ] Step 4: Implement `Initialize` — register actions, call `EnableConcurrentExecution()`
- [ ] Step 5: (Optional) Create `CodeFixProvider` with `[ExportCodeFixProvider]` + `[Shared]`
- [ ] Step 6: Add tests in `tests/Modulus.Analyzers.Tests/`
- [ ] Step 7: Run `dotnet test Modulus.slnx --filter "FullyQualifiedName~Analyzers"` — iterate until passing

### Analyzer Skeleton

```csharp
[DiagnosticAnalyzer(LanguageNames.CSharp)]
public sealed class MyAnalyzer : DiagnosticAnalyzer
{
    public override ImmutableArray<DiagnosticDescriptor> SupportedDiagnostics =>
        ImmutableArray.Create(DiagnosticDescriptors.MOD006);

    public override void Initialize(AnalysisContext context)
    {
        context.ConfigureGeneratedCodeAnalysis(GeneratedCodeAnalysisFlags.None);
        context.EnableConcurrentExecution();

        context.RegisterSyntaxNodeAction(
            AnalyzeNode,
            SyntaxKind.ClassDeclaration);
    }

    private static void AnalyzeNode(SyntaxNodeAnalysisContext ctx)
    {
        var node = (ClassDeclarationSyntax)ctx.Node;
        // ... semantic analysis
        ctx.ReportDiagnostic(
            Diagnostic.Create(DiagnosticDescriptors.MOD006, node.Identifier.GetLocation()));
    }
}
```

### Code Fix Skeleton

```csharp
[ExportCodeFixProvider(LanguageNames.CSharp, Name = nameof(MyCodeFixProvider))]
[Shared]
public sealed class MyCodeFixProvider : CodeFixProvider
{
    public override ImmutableArray<string> FixableDiagnosticIds =>
        ImmutableArray.Create("MOD006");

    public override FixAllProvider GetFixAllProvider() =>
        WellKnownFixAllProviders.BatchFixer;

    public override async Task RegisterCodeFixesAsync(CodeFixContext context)
    {
        var root = await context.Document.GetSyntaxRootAsync(context.CancellationToken);
        var node = root?.FindNode(context.Diagnostics[0].Location.SourceSpan);
        if (node is null) return;

        context.RegisterCodeFix(
            CodeAction.Create(
                title: "Fix: ...",
                createChangedDocument: ct => ApplyFix(context.Document, node, ct),
                equivalenceKey: "MyFix"),
            context.Diagnostics[0]);
    }

    private static async Task<Document> ApplyFix(
        Document document, SyntaxNode node, CancellationToken ct)
    {
        var editor = await DocumentEditor.CreateAsync(document, ct);
        // Use editor.ReplaceNode / editor.RemoveNode / editor.InsertAfter
        return editor.GetChangedDocument();
    }
}
```

## Testing Generators

Tests live in `tests/Modulus.Generators.Tests/`. Use `GeneratorTestHelper` (in `Helpers/`) — it builds a compilation with proper references and runs the generator.

```csharp
[Fact]
public void HandlerRegistration_DiscoversSingleCommandHandler()
{
    var result = GeneratorTestHelper.RunHandlerRegistrationGenerator("""
        using Modulus.Mediator.Abstractions;
        using Modulus.Mediator.Abstractions.Results;

        public record MyCommand : ICommand;

        public class MyHandler : ICommandHandler<MyCommand>
        {
            public Task<Result> Handle(MyCommand cmd, CancellationToken ct)
                => Task.FromResult(Result.Success());
        }
        """);

    result.Diagnostics.ShouldBeEmpty();

    var generated = result.GetGeneratedSource("ModulusHandlerRegistrations.g.cs");
    generated.ShouldContain("AddScoped<ICommandHandler<MyCommand>, MyHandler>()");
}

[Fact]
public void StronglyTypedId_MissingPartial_EmitsMODGEN001()
{
    var result = GeneratorTestHelper.RunGenerator("""
        [StronglyTypedId]
        public record struct ProductId;  // missing partial
        """);

    result.Diagnostics.ShouldContain(d => d.Id == "MODGEN001");
    result.GeneratedSources.ShouldBeEmpty();
}
```

**Feedback loop:**
1. Write test with source string
2. Run: `dotnet test Modulus.slnx --filter "FullyQualifiedName~Generators"`
3. If generated source doesn't match, print `result.GetGeneratedSource(...)` and inspect
4. Fix generator, repeat

## Testing Analyzers and Code Fixes

Tests live in `tests/Modulus.Analyzers.Tests/`. Use `AnalyzerTestHelper`.

```csharp
[Fact]
public async Task MOD003_ThrowInHandler_ReportsDiagnostic()
{
    var compilation = AnalyzerTestHelper.CreateCompilation("""
        public class Handler : ICommandHandler<Cmd>
        {
            public Task<Result> Handle(Cmd cmd, CancellationToken ct)
            {
                throw new NotFoundException("not found");
            }
        }
        """);

    var diagnostics = await compilation.GetDiagnosticsAsync(
        new ExceptionThrowingInHandlerAnalyzer());

    diagnostics.ShouldHaveSingleItem();
    diagnostics[0].Id.ShouldBe("MOD003");
}

[Fact]
public async Task MOD003_CodeFix_ReplacesThrowWithReturnError()
{
    const string source = """
        public class Handler : ICommandHandler<Cmd>
        {
            public Task<Result> Handle(Cmd cmd, CancellationToken ct)
                => throw new NotFoundException("msg");
        }
        """;

    var fixed = await AnalyzerTestHelper.ApplyCodeFixAsync(
        source,
        new ExceptionThrowingInHandlerAnalyzer(),
        new ExceptionThrowingInHandlerCodeFixProvider());

    fixed.ShouldContain("return Error.NotFound(");
    fixed.ShouldNotContain("throw new");
}
```

**Cross-module analyzer test (MOD001):**

```csharp
[Fact]
public async Task MOD001_DirectCrossModuleReference_ReportsError()
{
    // Build the "other module" assembly
    var otherAssembly = AnalyzerTestHelper.CreateCompilationWithReference(
        assemblyName: "Acme.Orders.Domain",
        source: "namespace Acme.Orders.Domain; public class Order { }");

    // Build the violating assembly
    var compilation = AnalyzerTestHelper.CreateCompilation(
        assemblyName: "Acme.Catalog.Application",
        source: "using Acme.Orders.Domain;",
        references: new[] { otherAssembly });

    var diagnostics = await compilation.GetDiagnosticsAsync(new ModuleBoundaryAnalyzer());
    diagnostics.ShouldContain(d => d.Id == "MOD001");
}
```

## Debugging Handler Discovery

When a handler isn't auto-registered after adding it:

1. **Rebuild** — source generators run at compile time: `dotnet build Modulus.slnx`
2. **Check the interface** — must be exactly one of: `ICommandHandler<T>`, `ICommandHandler<T,R>`, `IQueryHandler<T,R>`, `IStreamQueryHandler<T,R>`, `IDomainEventHandler<T>`, `IIntegrationEventHandler<T>`, or inherit `AbstractValidator<T>`
3. **Check for `abstract`** — abstract classes are excluded from discovery
4. **Check for open generics** — `MyHandler<T>` emits `MODGEN003` and is skipped
5. **Inspect generated file** — in IDE, look for `ModulusHandlerRegistrations.g.cs` under Analyzers > Source Generators
6. **Check assembly reference** — the generator only scans the assembly it's running in; cross-project handlers need the generator referenced in each project

```powershell
# Run only generator tests to verify discovery
dotnet test Modulus.slnx --filter "FullyQualifiedName~Generators"

# Run only analyzer tests
dotnet test Modulus.slnx --filter "FullyQualifiedName~Analyzers"
```

## Diagnostics Quick Reference

### Generator Diagnostics (MODGEN)

| ID | Severity | Trigger | Action |
|----|----------|---------|--------|
| MODGEN001 | Error | `[StronglyTypedId]` on non-partial type | Add `partial` keyword |
| MODGEN002 | Error | `[StronglyTypedId]` on non-record-struct | Change to `partial record struct` |
| MODGEN003 | Info | Open generic handler (e.g., `Handler<T>`) | Register manually if needed |
| MODGEN004 | Warning | `IModuleRegistration` impl missing `ConfigureServices` or `ConfigureEndpoints` | Add missing static method |

### Analyzer Diagnostics (MOD)

| ID | Severity | Fix | Scope |
|----|----------|-----|-------|
| MOD001 | Error | No | All assemblies with `{Prefix}.{Module}.{Layer}` naming |
| MOD002 | Warning | No | `ICommandHandler<>` / `IQueryHandler<>` implementations |
| MOD003 | Warning | Yes | `throw new DomainException()` inside handler Handle methods |
| MOD004 | Warning | Yes | EF Core / JSON attributes in `.Domain` assemblies only |
| MOD005 | Info | Yes | Public setter on `Entity` / `AggregateRoot` properties |

**Suppress a diagnostic when necessary:**
```csharp
#pragma warning disable MOD001 // intentional cross-module reference
using Other.Module.Domain;
#pragma warning restore MOD001
```
